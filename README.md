## 🎯 项目介绍

TestForge 是一款专为开发者设计的**轻量级、零依赖**智能测试用例生成与质量评估工具。它能够自动分析代码结构，生成高质量的测试用例，并提供多维度的测试质量评估报告。

### ✨ 核心价值

- 🔍 **智能分析** - 基于AST技术深度解析代码结构，精准识别可测试单元
- ⚡ **一键生成** - 自动生成符合pytest/unittest规范的测试代码
- 📊 **质量评估** - 多维度评估测试覆盖率、断言质量、代码结构
- 🎨 **友好界面** - 彩色TUI仪表盘，操作直观便捷
- 🌐 **多语言支持** - 支持Python/JavaScript/TypeScript
- 🔒 **零依赖设计** - 仅需Python 3.8+，无需安装任何第三方包

### 🚀 自研差异化亮点

1. **更轻量** - 相比现有测试工具，TestForge体积更小、启动更快
2. **更智能** - 内置代码复杂度分析，优先测试关键路径
3. **更全面** - 不仅生成测试，还提供质量评估和改进建议
4. **更易用** - 支持TUI交互界面，一问一答式操作

## 📦 核心特性

### 1️⃣ 代码分析器 (Code Analyzer)

- 🔬 基于AST的深度代码解析
- 📈 自动计算函数复杂度
- 🎯 识别可测试的函数、类、方法
- 📋 生成详细的代码结构报告

### 2️⃣ 测试生成器 (Test Generator)

- 🧬 自动生成单元测试用例
- 📝 支持pytest和unittest框架
- 🎨 生成参数化测试用例
- ⚡ 自动处理边界情况

### 3️⃣ 质量评估器 (Quality Assessor)

- 📊 七维度质量评估体系
- 🎯 覆盖率评分
- 💡 智能改进建议
- 📈 历史趋势追踪

### 4️⃣ TUI仪表盘 (Dashboard)

- 🎨 彩色终端界面
- 📊 实时数据可视化
- ⌨️ 键盘导航操作
- 📈 综合分析视图

## 🚀 快速开始

### 📋 环境要求

- Python 3.8 或更高版本
- 支持操作系统: Windows / macOS / Linux

### ⚡ 安装方式

#### 方式一：pip安装 (推荐)

```bash
pip install testforge
```

#### 方式二：从源码安装

```bash
git clone https://github.com/gitstq/TestForge.git
cd TestForge
pip install -e .
```

#### 方式三：一键安装脚本

```bash
curl -fsSL https://raw.githubusercontent.com/gitstq/TestForge/main/install.sh | bash
```

### 🎯 快速使用

#### 命令行使用

```bash
# 分析代码结构
testforge analyze ./src

# 生成测试用例
testforge generate ./src -o ./tests

# 评估测试质量
testforge assess ./tests

# 启动TUI界面
testforge dashboard
```

#### TUI界面使用

```bash
# 启动交互式界面
testforge

# 选择操作:
# 1. 分析代码
# 2. 生成测试
# 3. 评估质量
# 4. 综合仪表盘
# 5. 帮助信息
# 6. 退出程序
```

### 生成测试

```bash
# 使用pytest框架生成测试
testforge generate ./src -o ./tests --framework pytest

# 使用unittest框架生成测试
testforge generate ./src -o ./tests --framework unittest

# 设置目标覆盖率
testforge generate ./src -o ./tests --coverage 90
```

**生成的测试文件示例：**

```python
#!/usr/bin/env python3
"""
自动生成的测试文件 - Generated by TestForge
生成时间: 2025-01-20 10:30:00
框架: pytest
"""

import pytest

def test_add_numbers():
    """测试 add_numbers 函数 - 单参数"""
    result = add_numbers(1, 2)
    assert result is not None

def test_add_numbers_edge_cases():
    """测试 add_numbers 函数 - 边界情况"""
    result = add_numbers(0, 0)
    assert result == 0
    
    result = add_numbers(-1, 1)
    assert result == 0
```

testforge assess ./tests --format json

## 💡 设计思路

### 🎯 设计理念

1. **极简主义** - 零外部依赖，降低使用门槛
2. **开发者友好** - 清晰的输出，友好的交互
3. **实用主义** - 解决真实问题，提供实际价值
4. **持续迭代** - 倾听社区反馈，不断优化改进

### 🔧 技术选型

- **语言**: Python 3.8+ (生态成熟，易于扩展)
- **代码分析**: Python AST模块 (标准库，无需安装)
- **界面**: 终端原生UI (无需图形库依赖)
- **测试框架**: pytest/unittest (业界标准)

## 📦 打包与部署

### 🐳 Docker部署

```dockerfile
FROM python:3.11-slim

WORKDIR /app
COPY . /app

RUN pip install testforge

CMD ["testforge", "dashboard"]
```

### ☸️ Kubernetes部署

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: testforge-config
data:
  config.yaml: |
    analyzer:
      languages:
        - python
        - javascript
    generator:
      default_framework: pytest
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: testforge
spec:
  replicas: 1
  selector:
    matchLabels:
      app: testforge
  template:
    spec:
      containers:
      - name: testforge
        image: testforge:latest
        command: ["testforge", "dashboard"]
```
